Checklist avant de considérer une version terminée.

Lis le fichier de version en cours et vérifie :

1. Tous les critères de succès sont remplis
2. Aucune issue bloquante ouverte pour cette version
3. etat-actuel.md est à jour
4. Les tests (manuels ou automatisés) passent
5. La documentation est à jour
6. Pas de console.log / code de debug restant
7. La branche de version est prête à merger dans main

Si tout est OK, propose le merge et la mise à jour de feuille-de-route.md.
Si des points manquent, liste-les avec les actions correctives.
Lance les corrections toi-même si elles sont techniques.
Demande-moi uniquement si c'est une décision produit.
