Lis le fichier de version spécifié (ou le plus récent dans docs/versions/).

Découpe-le en tâches atomiques exécutables.

Pour chaque tâche :
- Titre court
- Ce qui doit être fait concrètement
- Fichiers à créer ou modifier
- Dépendances (tâches qui doivent être faites avant)
- Validation humaine nécessaire ? (oui/non + pourquoi)
- Points de blocage potentiels
- Complexité estimée (S/M/L)

Donne l'ordre logique d'exécution.
Prépare la première tâche dans docs/tache-en-cours.md.
Crée les issues GitHub correspondantes.
